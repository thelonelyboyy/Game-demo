# 万劫求仙 —— 通往「杀戮尖塔级成品」的分阶段优化方案

> 目标：把当前的可玩 Demo，打磨成一款**内容完整、数值耐玩、表现统一、可发布**的国风修仙卡牌肉鸽，质量对标《杀戮尖塔》。
> 用法：每个阶段都有「目标境地 / 任务清单 / 验收标准（Done 的定义）/ 预估」。**按阶段顺序推进**，前一阶段的验收标准不达标，不进入下一阶段。

---

## 一、现状评估（2026-06 基线）

## 阶段 0 · 工程质量地基（必须最先做）

**目标境地**：任何改动都能被自动验证；核心逻辑有单测护栏；存档不再因资源变动而损坏；从此「改东西不心慌」。

### 任务清单
1. **接入 CI（GitHub Actions）**
   - 新建 `.github/workflows/ci.yml`：在 push / PR 时，缓存 Godot 4.5.2，依次跑 `python scripts/validate_project.py` 与 `python scripts/run_godot_checks.py`。
   - 失败即红；为 `main`/`master` 加分支保护：CI 绿才可合并。
2. **修复 smoke 偶发失败**
   - 定位 `run-flow` 的「Lambda capture freed」根因（异步 tween/timer 在节点释放后回调）。在涉及的回调里加 `is_instance_valid()` 守卫，或改用 `await` 顺序化。目标：连续 20 次 smoke 0 失败。
3. **引入单元测试框架（GUT 或 GdUnit4）**
   - 覆盖纯逻辑：`CardEffect` 各效果结算、`ModifierHandler` 加成/百分比叠加、灵根缩放、`SaveGame` 迁移与自愈、章节缩放倍率、剑意/能量上限。
   - 首批 ≥ 40 个用例，纳入 CI。
4. **修 `validate_project.py` 的正则盲区**
   - 现有 `RES_PATH_RE` 漏掉部分 `preload("res://…")`（本轮已踩坑：`map_elite` 删除未被检出）。改为稳健匹配并补测：能检出「被引用但已删除」的资源。
5. **存档架构去脆弱化**（中期收益最大）
   - 现状：`SaveGame` 把整张对象图（卡牌/贴图/脚本路径）序列化，任一资源移动即损坏（已加自愈降级兜底）。
   - 改造：存档只存**最小可重建数据**——卡牌存 `id`+突破/费用/元素等变量，遗物存 `id`，角色存 `id`+三围，地图存种子+布局枚举。加载时按 id 从资源库重建对象图。
   - 加 `save_version` 字段 + 迁移注册表。

### 验收标准（Done）
- CI 在每次 push 自动运行，`validate + 7 项 smoke + 单测` 全绿才允许合并。
- smoke 连续 20 次无 flaky。
- 单测 ≥ 40 用例并通过；核心结算/数学/存档均被覆盖。
- 把任意一张被引用的卡/图标删除 → `validate` 立即报错。
- 移动/重命名任意美术资源后，旧存档仍能继续游戏（不再仅靠「清档降级」）。

## 阶段 1 · 战斗手感与核心系统

**目标境地**：单场战斗「看得懂、点得爽、读得准」，达到 STS 级别的信息清晰度与打击反馈。

### 任务清单
1. **出牌与瞄准反馈**：手牌 hover 放大、可出牌高亮/费用不足置灰；攻击牌拖拽出指向敌人的曲线 + 命中预览（结算后的实际伤害，已计破绽/护体）。
2. **意图系统补全**：本轮已做彩色气泡 + 攻击/护体贴图。补齐其余分类的正式图标（强化/诅咒/蓄力/召唤/回血/未知），并让数值实时反映修正（如被破绽放大）。
3. **状态/词条可读性**：状态图标统一尺寸 + 悬停说明；可叠加与持续型在 UI 上区分（破绽现为固定 +50% 不叠层，需明确表达）。
4. **打击感（juice）**：受击顿帧、伤害飘字、暴击/斩杀强化、出牌音画反馈、回合开始/结束转场。（注：玩家受击表现已按需求移除，需要时以「轻量化」方式重做）。
5. **高级效果类型扩展**：补「计数器/连锁/条件触发/每张同名牌+X/弃牌堆联动」等效果脚本，给构筑深度留空间。

### 验收标准
- 任意一张攻击牌，玩家在出牌前能看到「最终命中数值」。
- 12 种意图全部有正式图标且数值随修正实时更新。
- 一场战斗的所有反馈（出牌、命中、获得护体、施加状态、击杀）都有视觉/音效。
- 新增 ≥ 6 种高级效果脚本并各有 ≥ 1 张卡使用、被单测覆盖。

## 阶段 2 · 内容厚度（卡牌 / 敌人 / Boss / 事件 / 遗物 / 融合）

**目标境地**：内容量级达到「一局 45–90 分钟、多周目不重复」的 STS 体量。

### 任务清单（量化目标）
1. **卡牌**：魔修已达到 **75 张**并具备自损血契 / 煞气魔焰 / 魂印引爆三条路线；其余职业仍需补到 ≥75。目标稀有度分布约 **白 45% / 蓝 32% / 金 18% / 暗金 5%**。
2. **通用卡**：扩到 ≥ 12 张，覆盖基础攻防与跨职业组件。
3. **敌人**：每章专属普通敌人 ≥ 8 种、精英 ≥ 3 种，三章合计 ≥ 33 普通 + ≥ 9 精英。敌人 AI 行为多样（蓄力、召唤、增益、阶段转换）。
4. **Boss**：每章 ≥ 3 个可轮换 Boss（合计 ≥ 9），每个 Boss 有**多阶段/独特机制**，而非纯数值堆叠。
5. **事件**：已达到 45 个，三章各 15 个互不重叠，并使用洗牌袋避免同章生成重复；已有分支后果、风险/收益抉择与魔修主题事件，后续补灵根专属条件分支。
6. **遗物**：当前奖励池 45 件，其中魔修专属 9 件、魔修可用首领法宝 6 件；已覆盖开战/回合/受击/自损/魂印消费/出牌/事件/篝火，四档稀有度、章节来源权重、商店分层定价和 Boss 首领档三选一均已完成，后续补到 ≥80 并继续实战校准分级。
7. **融合**：已从实际 4 个固定配方扩到 20 个，其中 16 个魔修丹方会合并双方主效果与生命周期触发；后续补独立融合插画并继续实战校准费用。

### 验收标准
- 各职业 ≥ 75 卡、各 draftable 池数量达标、`validate_project.py` 内容门槛全过。
- 每章敌人/Boss 阵容独立可配置（见阶段 3 的「分章敌池」）。
- 同章普通/精英/Boss 各自使用加权洗牌袋，首轮耗尽前重复率为 0；后续人工三周目继续校准各权重带来的出现顺序。
- 每个 Boss 都能被至少 2 种不同构筑稳定击败（设计自检）。

## 阶段 3 · 数值平衡与难度曲线

**目标境地**：难度平滑上升、构筑有强弱但无「唯一解/废牌堆」；提供可重复挑战的难度阶梯。

### 任务清单
1. **分章敌池（遭遇数量基线已完成）**：`BattleStats` 已有章节范围，普通/精英/Boss 按章节筛选；现为每章 8 普通/3 精英，`content-progression` smoke 验证三章池互不重复。下一步增加独立敌种，减少复用美术与 AI。
2. **Boss 梯队（数量基线已完成）**：三章均达到 3 名 Boss 随机轮换，9 名 Boss 全部配置二阶段与独立固定序列；后续重点替换复用美术并做多构筑胜率校准。
3. **经济与曲线**：法宝/消耗品已按章节和来源分层，卡牌奖励已有章节稀有曲线与连续 6 次未见高稀有的硬保底；下一步用模拟/表格核对每章金币产出 vs 商店/移牌成本，保证「该买什么」有取舍张力。
4. **难度等级（首版已完成）**：心魔 0–15 已实现累计修正、独立进度档、逐级解锁、选人入口和结算反馈；后续用自动模拟与人工试玩校准各级通关率，并补按职业独立记录。
5. **平衡工具化**：写一个 headless 自动对战脚本（用启发式 AI 跑 N 局），输出每职业/每难度的通关率、关键卡使用率，作为平衡回归基线。

### 验收标准
- 三章敌人阵容互不相同；Boss 难度单调递增、无忽难忽易。
- 中等难度：新手首通率约 15–25%，熟练玩家约 60–70%（自动对战 + 真人抽样验证）。
- 无单卡使用率 > 80%（避免唯一解）、无整类卡使用率 < 2%（避免废牌）。
- 难度等级 1→15 可玩且记录独立。

## 阶段 4 · UI/UX 全面打磨

**目标境地**：每个界面在桌面与移动分辨率下都清晰、自适应、风格统一；信息密度与可读性对标 STS。

### 任务清单
1. **战斗 HUD**：落地本轮草图方向（玩家立左、群魔立右、暗金墨色、扇形手牌、清晰意图）。
2. **地图**：路径可读性、当前/可达/已访问节点状态分明（已加卷轴底板，需做不同分辨率适配与缩放）。
3. **牌库/图鉴/奖励/商店/篝火/融合**：统一卡牌显示组件（复用 `card_visuals`/`card_menu_ui`），消除重复布局代码（本轮已识别 6+ 处重复，建议抽 `CardGridUI`）。
4. **设置页**：音量（主/乐/效）、分辨率/窗口、语言、重置存档、键位（若支持）。
5. **可达性**：字号档位、色盲友好的意图/元素配色、关键信息不只靠颜色区分。
6. **响应式**：项目当前为 Mobile 渲染；所有界面在 16:9 与竖屏/宽屏都不溢出不遮挡。

### 验收标准
- 所有界面在 1280×720 / 1920×1080 / 常见移动比例下无溢出、无遮挡、可操作。
- 抽出统一卡牌网格组件后，奖励/删除/升级/融合/商店/牌库共用一套，代码重复消除。
- 设置项可用且即时生效、可持久化。
- 一次「全界面视觉 QA」清单逐项通过。

## 阶段 5 · 美术统一与替换

**目标境地**：消灭占位图，全项目一套统一画风（云海仙山 / 暗金墨色 / 实体卡质感），达到上架级视觉。

### 任务清单
1. **替换占位**：本轮审查出 ≤32px 的占位（`tiles/` 像素图、低清 HUD 图标）需全部替换；删除未用资源（已做一批）。
2. **卡牌插画**：为高频卡先行补正式插画，保持 `card_style` 卡框布局坐标不变（贴图优先、命名规范）。
3. **角色帧动画**：本轮魔修已接（待机/攻击/受击）。补齐剑/体/驭三系，并增加 **死亡** 动画；用 `battle_anim_id` 机制即插即用。
4. **敌人/Boss 美术与动画**：至少待机 + 攻击 + 受击 + 死亡。
5. **标题与品牌**：替换主菜单标题图（当前仍是旧「百劫成仙」图，文件 `baijie_chengxian_title.png`）；统一 icon、字体（当前主题用系统字，建议引入正式国风字体）。
6. **特效（VFX）**：出牌、命中、状态、灵根共鸣、合炼等关键时刻的粒子/着色器表现。

### 验收标准
- 项目内无 ≤32px 占位视觉资源；`validate` 无未用/缺失资源。
- 四职业均有 standby/attack/attacked/death 动画；主要敌人/Boss 同标准。
- 标题、icon、字体、卡框、背景风格统一；随机截 10 张界面图风格一致。

## 阶段 6 · 音频

**目标境地**：声音覆盖每个关键动作与场景，有沉浸感且可调。

### 任务清单
1. **BGM**：主菜单 / 地图 / 普通战 / 精英 / Boss / 商店 / 事件 各有曲目，带淡入淡出与循环。
2. **SFX 覆盖**：出牌、命中（按元素/类型区分）、护体、状态、抽牌、金币、升级、合炼、胜负。
3. **音频总线与设置**：主/乐/效三条总线，接入设置页音量；静音不影响逻辑。

### 验收标准
- 每个场景有对应 BGM 且切换平滑；每个关键动作有 SFX。
- 音量设置即时生效、可持久化；全静音下游戏可正常完成一局。

## 阶段 7 · 元进度与可重复性

**目标境地**：玩家有长期目标，愿意一局接一局。

### 任务清单
1. **解锁系统**：初始仅开放部分卡/遗物，随通关逐步解锁，丰富 draft 池。
2. **难度阶梯**：阶段 3 的爬塔/心魔等级，逐级解锁与独立记录。
3. **成就 / 统计**：通关记录、最高难度、各职业胜率、单局最高伤害等。
4. **（可选）每日挑战 / 自定义种子**：固定种子的可分享挑战。

### 验收标准
- 新档到全解锁有清晰、可达成的进度路径。
- 成就/统计持久化且正确；难度等级逐级解锁可用。

## 阶段 8 · 新手引导、本地化与可达性

**目标境地**：新玩家 5 分钟内学会核心循环；具备多语言能力。

### 任务清单
1. **新手引导**：首局分步提示（出牌/费用/意图/结束回合/地图/灵根），可跳过。
2. **本地化（i18n）**：把所有 UI/卡牌/事件文本抽到翻译表（Godot CSV/翻译），先中文，预留英文。
3. **图鉴即帮助**：关键词（破绽/魂印/铸剑/灵根…）可点击查看说明。

### 验收标准
- 新玩家无需外部说明即可完成首局核心操作。
- 文本 100% 走翻译表，切换语言全界面无硬编码残留。

## 阶段 9 · 性能、平台适配与存档可靠性

**目标境地**：目标平台稳定 60fps、无内存泄漏、存档读档稳如磐石。

### 任务清单
1. **性能**：战斗高频实例（卡牌/特效）用对象池；清理 smoke 里提示的资源泄漏告警；分辨率/质量档位。
2. **平台适配**：明确首发平台（PC/移动）；输入（鼠标/触控/手柄）、安全区、退出/暂停行为。
3. **存档可靠性收尾**：基于阶段 0 的重建式存档，做断电/损坏/跨版本的健壮性测试与自动备份。

### 验收标准
- 目标分辨率下战斗稳定 60fps；退出时无泄漏告警。
- 触控/鼠标（按平台）全流程可玩。
- 杀进程/改版本/损坏存档三种情况都能安全恢复或干净重来。

## 阶段 10 · 发布准备

**目标境地**：可对外发布的构建 + 商店素材 + 质量保障流程。

### 任务清单
1. **打包流水线**：各平台导出预设 + CI 出包（标签触发自动构建产物）。
2. **QA 流程**：发布前回归清单（全职业 × 多难度通关、全房间类型、存读档、设置项、本地化）。
3. **商店素材**：截图、预告、icon、商店页文案；Demo→正式版的内容门控。
4. **崩溃/分析**：基础崩溃上报与匿名数据（通关率/弃用卡），用于上线后调优。

## 二、推进节奏与依赖关系

```
阶段0(地基) ──→ 阶段1(手感) ──→ 阶段2(内容) ──→ 阶段3(平衡)
                                  │                 │
                                  └──→ 阶段4(UI) ──→ 阶段5(美术) ──→ 阶段6(音频)
                                                          │
阶段7(元进度) ── 阶段8(引导/本地化) ── 阶段9(性能/平台) ── 阶段10(发布)
```

- **必须先做阶段 0**：没有 CI/单测/稳健存档，后面越堆越脆。
- 内容（2）与平衡（3）交替迭代：每补一批内容就跑一次平衡回归。
- 美术（5）/音频（6）可与内容并行，但要在 UI（4）定型后再大规模产出，避免返工。
- 元进度（7）之后的阶段面向「上线」，可在内容/平衡基本定型后启动。

## 三、近期最高优先级（建议从这里开始）
1. **阶段 0 全部**（CI + 单测 + 修 flaky + 修 validator + 存档重建）——一次性把地基打牢。
2. **阶段 3 的分章敌池 + Boss 梯队**——成本低、立刻改善难度体验。
3. **阶段 1 的出牌命中预览 + 意图图标补全**——直接提升战斗手感。
4. **阶段 5 的占位替换（高频卡插画 + 标题图）**——视觉第一印象。

> 每完成一个阶段，更新本文件的勾选状态，并在 `HANDOFF.md` 记录当前进度。
