# 一、项目概述与设计愿景

## 1.1 一句话定位

《万劫求仙》是一款**国产修仙题材的杀戮尖塔式卡牌肉鸽**：玩家扮演一名入劫域求仙的修士，在三章爬塔中构筑牌组、凝聚灵根、搜集法宝，最终斩杀三尊劫主飞升——或者死在半路，把这一世的机缘留给下一世。

技术底座为 **Godot 4.5.2 stable mono**，单机、离线、键鼠操作，目标分辨率 1920×1080（HUD 按 `REFERENCE_SIZE` 等比缩放，缩放系数钳制在 0.78–1.12）。

## 1.2 参考系与差异化

本作的玩法骨架直接对标《杀戮尖塔》（Slay the Spire，下称 STS），并在开发过程中系统性地研究了 STS2 的 606 张卡牌数据（见 `杀戮尖塔2_卡牌数据_1.xlsx` 分析结论，已内化到本作的卡池预算模型中）。与 STS 的差异化主要在三个方向：

1. **题材语言的彻底汉化**：不是贴皮换字，而是把 STS 的机制词汇整体翻译进修仙语境——能量是"灵力"，药水是"符箓丹药"，遗物是"法宝"，Neow 祝福是"劫中遗赠"，升级卡牌叫"突破"，删卡叫"炼化"。所有系统命名服从同一套世界观词典。
2. **职业机制的多资源耦合**：当前主角魔修同时运转三套资源系统（煞气/魂印/魔焰焰轮），复杂度定位对标 STS2 中最复杂的"亡灵契约师"范式，而非单核职业。设计赌注是：多资源耦合带来的构筑深度足以支撑单职业 Demo 的重复可玩性。
3. **灵根系统作为跨职业的第二构筑轴**：开局三选一的五行灵根、"同元素牌数量"驱动的三段成长、圆满解锁的每回合被动——它给"拿哪张牌"这个 STS 核心决策叠加了一层元素维度的考量，是本作原创度最高的系统。

## 1.3 核心循环

```
主菜单 → 选人（魔修）→ 灵根三选一 → 进入第一章地图
  ↓
地图选路（13 层 / 章）：普通战 / 精英战 / 事件 / 商店 / 宝箱 / 篝火 / 祝福 → Boss
  ↓
战斗循环：抽 5 张 → 4 灵力出牌（三系统联动）→ 结束回合 → 敌人按意图行动 → 循环
  ↓
战斗胜利 → 灵石 + 三选一卡牌 +（概率）符箓丹药 →回到地图
  ↓
章 Boss 击杀 → 回满血 → 下一章（敌人数值按章节倍率爬升）
  ↓
第三章 Boss 击杀 → 胜利结算 / 任意时刻死亡 → 失败遗产（下一世可继承一件法宝）
```

**单局时长目标**：完整三章 40–60 分钟；单场普通战 2–3 个玩家回合，精英 4–6，Boss 6–10。

## 1.4 三大体验支柱

整个开发与优化过程围绕三根支柱展开，所有取舍最终都回到这三个问题上：

**支柱一：构筑有深度（Build Depth）**
玩家在第 3 层做的拿牌决定，应该在第 20 层的 Boss 战里表现为完全不同的打法。魔修三系统 + 灵根元素轴 + 法宝协同，理论构筑空间：煞气自损流 / 魂印引爆流 / 魔焰七色流 / 元素灵根特化流，以及它们的杂交。

**支柱二：打击有手感（Game Feel）**
2026-07-06 的"战斗手感优化专场"一次性落地约 25 项反馈（详见第四、五章），之后又追加三轮调速与醒目化。设计立场：**手感不是装饰，是信息**——每一次掉血、格挡、状态变化都必须在 300ms 内以视觉+听觉双通道告知玩家，且多个反馈的节奏刻意错开避免机械感。

**支柱三：氛围有辨识度（Dark Guofeng）**
"暗黑修仙"不是暗色滤镜：黑金主色 + 暗紫点缀 + 水墨质感面板（InkTheme 设计系统），配 CC0 暗黑管弦/交响金属 BGM 与锣、钟、书页等国风音效重音。视觉参考锚点是 `art/design/target.png` 系列目标图。

# 二、核心玩法系统设计

## 2.1 卡牌系统

### 2.1.1 卡牌属性模型

每张卡是一个 `Card` 资源（`custom_resources/card.gd`），关键字段：

- **type**：攻击（ATTACK）/ 技能（SKILL）/ 功法（POWER）。功法即 STS 的能力牌——打出后永久生效、不进弃牌堆。
- **rarity**：白（COMMON）/ 蓝（UNCOMMON）/ 金（RARE）/ 暗金（MYTHIC）。四档稀有度对应商店定价 90/140/210/320 灵石（±10% 浮动）与奖励权重。暗金卡在商店以 4% 概率上架（`MYTHIC_SHOP_CHANCE`），是"逛店惊喜"的顶层设计。
- **target**：自身 / 单体敌人 / 全体敌人 / 所有人。单体攻击牌走瞄准弧线交互，其余拖出手牌区自动释放。
- **cost + element + profession + mechanic_tags**：费用、五行元素（灵根系统的挂钩点）、职业归属、机制标签（如"血炼""铸剑"，供 `class_mechanic_handler` 侦听）。
- **exhausts**：消耗标记。打出后彻底离场，不入弃牌堆。
- **is_temporary（临时牌）**：英雄技能与"发现"机制生成的牌。**用完或者不用都不会进入弃牌/抽牌堆循环**——打出即消散，回合结束若仍在手牌也直接消散。这是与"消耗"的关键区别：消耗牌不打就留在循环里，临时牌是真正的过客。

### 2.1.2 效果系统：configured_effects 架构

**历史决策**：早期卡牌用字段驱动（`base_damage`、`muscle_stacks` 等散字段），每加一种效果就要改卡牌基类。2026-06-16 彻底删除该路径，全部效果统一为 `configured_effects` 数组——每个效果是一个独立的 `CardEffect` 子类资源，卡牌只是效果的容器。

`CardEffect` 基类（`custom_resources/effects/card_effect.gd`）提供四个通用维度：

1. **数值（amount）+ 灵根缩放开关（affected_by_spirit_root）**：数值型效果默认吃灵根加成，特殊效果可关闭。
2. **条件系统（ConditionType）**：无条件 / 本牌元素与灵根一致 / 拥有机制标签 / 卡牌类型匹配 / 玩家拥有指定状态 → 满足则叠加 `bonus_amount`。这套条件让"火灵根时+3伤害"这类卡不需要专用代码。
3. **目标改写（TargetMode）**：效果可以无视卡牌目标重定向到玩家自身/全体敌人/所有人——单张卡因此可以同时打敌方和给自己上 buff。
4. **描述模板（description_template）**：`{amount}` 占位符自动代入修正后数值，卡面描述与实际数值永不脱节。

特殊效果类（魂印引爆、魔焰 rider、X 费自损、发现等）各自继承扩展，其专用字段**刻意排除在 Excel 数值管线之外**（见 5.3），防止表格误编辑破坏机制。

### 2.1.3 卡池预算模型（源自 STS2 数据分析）

对 STS2 606 张卡的分析得出其"工业化配比"：每职业 87–89 张，稀有度配额几乎逐职业复刻（罕见 36/稀有 26/普通 20），类型比约攻击 35%/技能 45%/能力 20%，1 费为主干，每职业固定 2 张 X 费。

魔修卡池按此模型做了一轮结构化补全（从 51 → 67 张）：补 10 张功法引擎卡（"每当你失血→""每当你引爆魂印→"类触发器）、4 张防御技能、2 张 X 费终结技，并给 4 张爆发牌补"消耗"标签。补全后类型比 45/36/19，落在"攻击前倾"档——刻意贴近 STS 铁甲战士的档位而非平均值，因为魔修的献祭身份需要攻击密度。

**设计规则（沉淀）**：
- 每职业必须有一个"专属命名资源"做主题锚点（魔修有三个：煞气/魂印/焰轮）。
- 功法（能力牌）占比 ~20%，它们是把普通动作转成引擎的"每当"触发器，是 build 成立的必要条件。
- X 费牌每职业恰好 2 张，作为费用曲线的顶端张力。

### 2.1.4 升级、融合与临时成长

- **突破（升级）**：篝火/祝福/事件三渠道。数值型 +50% 向上取整（`_upgrade_number`），或降费型。名称加"+"后缀。
- **融合**：`fusion_recipes/` 配方制——两张特定卡合成一张融合卡（fusion_level 标记），精英战奖励提供融合机会。这是本作独有的卡牌减法机制：融合同时瘦身牌组并产出更强的单卡。
- **临时降费**：蓝·灵涌等效果的 `reduce_cost_for_turn` 只持续当回合，回合结束 `reset_temporary_cost` 归还。

## 2.2 魔修职业：三系统耦合设计

魔修是 Demo 唯一开放职业，其机制野心是"多资源耦合"——三套系统各自成立，交叉处产生构筑火花。引爆倍率全局常数 X=3。

### 2.2.1 系统一：煞气（Sha Qi）——风险资源

- **获取**：受到伤害（`player_hit`）+1；出煞气牌（引煞诀/聚煞）；功法"养煞"回合末+1。
- **阈值档位**（`class_mechanic_handler` 用 modifier 原地改值实现，刻意不用 remove_value 以规避 queue_free 延迟丢值的坑）：
  - **≥3（凝聚）**：卡牌伤害 +1（FLAT 修正）
  - **≥6（翻涌）**：造成与受到的伤害 ×2（双刃！）
  - **≥10（天魔降世）**：本回合伤害 ×3，回合末关闭，下回合开始付出代价——失去 50% 最大生命、煞气降至 5。
- **设计意图**：煞气是"挨打变强"的滚雪球资源，但 ×2 档同时放大受伤，天魔降世是明码标价的自杀式爆发。玩家管理煞气的过程就是在管理风险敞口。
- **可视化**：玩家信息卡上方常驻"煞气 N"徽章（按档位灰→橙→红→亮红变色发光、层数变化 punch 弹跳）；跨档时中央横幅宣告（"煞气翻涌 · 造成与受到伤害 ×2"）+ 锣声重音 + 玩家立绘按档位渐进染红作常驻气场。**机制可见性是本系统迭代最多的部分**——数值藏在 modifier 里玩家看不见，三层可视化（常驻徽章/瞬时横幅/持续气场）分别回答"现在几层""刚跨档了""大致处于什么状态"三个问题。

### 2.2.2 系统二：魂印引爆（Soul Mark）——延迟结算资源

- **上印**：多张卡给敌人叠魂印层数（噬魂爪/灯影噬魂/万魂牢等）。
- **两种结算路径**：
  1. **回合末 DoT**：敌人自身状态结算，每层 2 伤，**不吃玩家增伤**（防止双重放大失控）。
  2. **主动引爆**（裂魂/三魂同焚/魂葬）：消耗 N 层 → 3×N 伤害，**走玩家 DMG_DEALT 增伤**——会被煞气 ×2/×3 放大。这是煞气×魂印的核心协同点：先堆印，等煞气翻涌的回合一波引爆。
- **转化路径**（魂铠/摄魂续元）：消耗魂印换护体/治疗——给上印流一个防御出口。
- **设计规则**：DoT 保底不吃增伤、引爆吃增伤，两条路径的数学差异决定了"攒不攒"的决策价值。

### 2.2.3 系统三：魔焰焰轮（Flame Wheel）——回合内组合资源

- **机制**：7 张七色魔焰卡（紫蚀魂/白护魂/绿镇煞/蓝灵涌/黑噬血/黄狂烬/红焚界）。打出魔焰卡时：先按焰轮中"**其它**颜色"的数量结算共鸣加成，再把本卡颜色点亮进焰轮。焰轮回合开始/结束清空。
- **七色 rider 各有专属逻辑**：绿消耗 2 煞气令本牌护体翻倍（煞气协同）；蓝给手牌魔焰卡降费（连锁引擎）；黄叠加本回合共鸣伤害加成；红在焰轮达 4 色时引爆所有敌人魂印（魂印协同、焰轮的终极 payoff）。
- **设计意图**：焰轮是"单回合内的顺序谜题"——同样五张魔焰，出牌顺序不同收益差数倍。它给魔修的操作层注入了 STS 没有的即时组合乐趣。
- **可视化**：战斗左下角常驻焰轮盘（7 色珠，点亮高亮、未点暗淡，魔修开局即显示空盘）；新点亮的色珠 1.7 倍放大回落 + 高亮闪烁（黑色魔焰有专门提亮兜底防看不见）。

### 2.2.4 三系统的交叉表

| 交叉点 | 协同内容 |
|---|---|
| 煞气 × 魂印 | 引爆伤害吃煞气倍率 → "攒印等翻涌"打法 |
| 煞气 × 焰轮 | 绿·镇煞消煞换双倍护体 → 煞气的防御性出口 |
| 焰轮 × 魂印 | 紫·蚀魂共鸣上印；红·焚界 4 色全场引爆 → 焰轮流的收尾炸弹 |
| 三系汇合 | 黄狂烬(共鸣增伤)+红焚界(引爆)+天魔降世(×3) 的理论一回合斩杀线 |

## 2.3 英雄技能系统（魔焰焚心）

**定位**：炉石"英雄技能"式的常驻主动能力，给魔修一个不依赖抽牌的稳定小决策点，同时充当三系统的润滑剂。

- **效果**：受到 2 点伤害，生成 1 张临时随机魔修卡（不含攻击与基础防御）加入手牌。临时牌打出即移除、未打出回合末消散，**永不进入抽牌/弃牌循环**。
- **限制**：每回合限用一次（`player_handler._hero_skill_used_this_turn` 逻辑层拦截 + UI 层经 `Events.hero_skill_used` 信号同步置灰按钮，双保险）。
- **代价语义（重要设计决定）**：代价走**真实伤害结算**（`player.take_damage(2, DMG_TAKEN)`）而非直扣血。连锁语义全部是有意为之：
  - 有 ≥2 护体时施放 → 护体吸收，等于免费——"先铺盾再发动"成为微技巧；
  - 煞气 ≥6 时 → 代价被 ×2 放大为 4 点——爆发期用技能更疼，风险自洽；
  - 掉血命中 → 触发 `player_hit` 照常叠 1 层煞气——技能自带煞气引擎；
  - 完整受击演出（飘字/音效/停顿/震动）。
- **UI**：按钮位于玩家信息卡正下方（156×76，与结束回合同款红金浮雕边框），右侧留白预留第二技能位（from-right (70,40)，154×76）。代价与规则只在悬停 tooltip 展示，界面保持干净。
- **扩展方向**：第二技能位已在布局上预留。候选设计——防御向（"魔躯：消耗 3 层煞气，获得等量×3护体"）或资源向（"引魂：本回合下一张魂印牌费用-1"），原则是与焚心形成"进攻/防守"或"发牌/用牌"的轴向互补。

## 2.4 灵根系统

### 2.4.1 机制全貌（2026-07-01 重做版）

- **开局**：五行（金木水火土）随机三选一。选定后：一张打击或防御转化为该元素 + 获得一张随机同元素"灵根职业卡"。
- **成长**：按牌组内**永久牌**的同元素数量分段——初悟（0–3 张）/ 小成（4–6 张）/ 圆满（7+ 张）。
- **数值缩放**（`character_stats.get_spirit_root_modified_value`）：初悟主数值+1；小成 ceil(×1.2)+1；圆满 ceil(×1.4)+1。
- **圆满被动**（每回合最多一次，`spirit_root_handler` 回合开始重置）：
  - **火**：每回合第一张火攻击牌弹出选择——伤害×1.5 或按最终实际伤害的 50% 溅射其它敌人（唯一带主动选择的灵根，火的"操作感"定位）；
  - **金**：第一张金牌 +1 劲气；
  - **水**：第一张水牌抽 1，抽到的第一张牌本回合费用 -1；
  - **土**：第一张土牌 +1 真元（护体增幅词条）；
  - **木**：回合结束若打出过木牌，回 3 血（满血则转 3 护体——解决了木灵根满血时废被动的旧问题）。
- **UI 反馈**：顶栏灵根徽章显示"元素 + 阶段 + 同元素张数"，tooltip 给出完整规则、当前进度与"距下一阶段还差 N 张"。

### 2.4.2 设计检讨与历史教训

旧版灵根（圆满门槛 10 张、被动只在圆满解锁）经过一轮完整的批判性分析后重做，沉淀出的教训写在这里防止回退：

1. **全有/全无的解锁曲线是死设计**——旧版绝大多数对局摸不到圆满，玩家前中期对灵根的感知只有干巴巴的 +1。新版把门槛降到 7 并让数值缩放全程在线。
2. **UI 文案与代码必须同源**——旧版 UI 写"+2/×1.8"而代码是"+1/×1.4"，玩家被系统性误导。现在缩放规则的展示文本直接由同一常数生成逻辑保证一致。
3. **构筑引导要给"能不能凑齐"的信息**——选灵根时玩家无法预知该元素卡的供给量。目前靠"开局送一张同元素职业卡"缓解，中期方案是在灵根选择界面显示当前职业卡池中各元素的可获取张数。

## 2.5 敌人与 AI 系统

### 2.5.1 三层数值架构

敌人强度 = **基线值 × 每场战斗倍率 × 章节倍率**，三层各司其职：

1. **基线**：写在动作脚本 `@export` 默认值（三套共享脚本覆盖 19 个普通敌人）或专属 AI 场景的覆盖值里。这是"这只怪的本色"。
2. **每场倍率**：`BattleStats.enemy_health/damage_multiplier`——同一只怪在精英战里可以更凶（精英战伤害倍率 1.15–1.35）。
3. **章节倍率**：`map_generator` 的 CHAPTER_*_MULTIPLIERS（血量 1.0/1.30/1.65，伤害 1.0/1.15/1.30，金币 1.0/1.20/1.40）——全局难度爬升，经 `_battle_for_room` 复制缩放、不污染战斗池模板。

**调平操作指引**：改单怪手感 → 动基线；改精英/Boss 压迫感 → 动每场倍率；改整体难度曲线 → 动章节倍率。三层不要交叉补偿。

### 2.5.2 行为系统

- **加权随机（默认）**：`EnemyActionPicker` 按 chance_weight 抽取动作，附带条件动作优先（如"低血时必逃跑"类）。
- **固定套路（fixed_sequence）**：动作子节点下标数组按回合循环，忽略权重。所有 Boss 与 4 个教学型敌人（符纸兵/雾隐狼/牛魔/渊狱剑魂）使用。**固定套路是"可学习的敌人语言"**——玩家背下循环后能提前规划，这是 STS Boss 设计的核心乐趣。
- **群体协作动作**（`enemies/shared/`，近期新增）：给队友格挡（block_allies）、给队友 buff（buff_allies）——多怪战不再是各自为战的单体堆，出现了"先杀奶妈还是先杀打手"的目标优先级决策。
- **状态卡植入**（近期新增的高级机制）：敌人把负面状态卡塞进玩家的抽牌堆/弃牌堆（add_card_to_draw/discard_action）。现有三张：**蚀日之痕**（蚀日妖皇蓄力时植入）、**心魔**（黑莲圣母的诅咒循环）、**幽冥契书**。这是 STS"衰弱化牌组"式的软压力——不打脸，但污染你的循环。设计规则：状态卡必须"无法打出"或带明确的解除手段，且在敌人意图气泡中可预读。

### 2.5.3 Boss 设计哲学：五种战斗节奏

每尊 Boss 的固定循环对应一种要求玩家改变打法的"考题"：

| Boss | 章节定位 | 循环 | 考什么 |
|---|---|---|---|
| **骨龙**（210 HP） | 猛攻型 | 双爪 2×11 → 龙息 18 → 双爪 → 骨甲 18 | 裸拼输出效率，防御不足直接被磨死 |
| **渊狱剑魂**（235 HP） | 节奏型 | 斩击 15 → 格挡 22 → 连击 2×9 → **处决 25** | 记循环、在处决回合前囤护体 |
| **黑莲圣母**（245 HP） | 成长+污染 | 心魔植入 → 莲刺 12（吃力量成长）→ 护体 22 → 莲刺 | 比 DPS 竞速，拖得越久莲刺越痛、牌组心魔越多 |
| **天阙镇将**（265 HP） | 镇守型 | 镇守 30 格挡 → 戟击 16 → 天雷 26 → 戟击 | 破盾期爆发管理，攒一波越过 30 格挡 |
| **蚀日妖皇**（290 HP） | 蓄力大招+污染 | 爪击 14 → 蓄势（格挡15+植入蚀日之痕）→ **蚀日之啮 34** | 三拍节奏应对，大招回合的 34 伤是全游戏最高单击 |

配套演出：Boss 战自动切换交响金属 BGM、开场 1.45s 后紫色名牌横幅 + 钟声、地图上 Boss 节点常驻红光脉冲——从地图到战斗的威慑感是连续的。

## 2.6 数值配平模型（100 血基准）

### 2.6.1 玩家侧吞吐

魔修：100 HP / 每回合 4 灵力 / 抽 5 张。基准单卡功率：1 费攻击 6–9 伤、1 费技能 5–8 护体。**每回合吞吐**：早期约 18–24 输出或 10–15 护体，成型期 30–40+。

### 2.6.2 敌人侧压力目标

| 战斗类型 | 时长目标 | 玩家血量代价目标 | 敌方配置基准 |
|---|---|---|---|
| 普通战 | 2–3 回合 | 10–20 HP | 单怪 HP 38–80 或双怪组，组伤 14–22/回合 |
| 精英战 | 4–6 回合 | 25–40 HP | HP 105–155，伤害倍率 1.15–1.35 |
| Boss 战 | 6–10 回合 | 35–55 HP | HP 210–290，伤害倍率 1.15–1.25 |

推导逻辑：修复资源（篝火回 30% 上限、章末回满、事件回 14–29、丹药）决定了每章可承受的总消耗约为 1.5–2 管血；13 层中约 6–8 场战斗，按上表分配正好收支紧张——"紧张但不绝望"。

### 2.6.3 一次系统性的再基线（2026-07 已完成）

血量从 30 提到 100、灵力 3 提到 4 之后，全部敌人做了 ×2.6–3.0 血量、×2.2–2.5 伤害的再基线（23 个敌人逐一定值，非无脑乘法——骨龙从荒谬的 40 血单独拉到 210）。事件数值同轮跟进（治疗 ×1.8、代价 ×1.6、金币 ×1.2）。**遗留**：以上为模型推导值，未经完整实战校验，需按体感微调（最快杠杆：三套共享脚本默认值 + 精英战 tres 倍率）。

## 2.7 经济系统

- **收入**：开局 70 灵石；普通战 ~金币基准 × 章节倍率；事件 30–110；祝福最高 250（付血代价）。
- **支出**：商店卡 90/140/210/320（按稀有度）±10%；法宝价位带更高；删卡服务；丹药。
- **设计张力**：单章总收入约 250–400，买一张金卡+一件法宝就见底——"这家店买什么"是真决策。优惠券法宝（CouponsRelic）与商店打折机制提供经济流的 build 方向。

## 2.8 祝福系统（劫中遗赠 = Neow 式开局祝福）

### 2.8.1 结构：跨来源四选一

数据外置 `data/blessings.json`（5 个"来源"× 各 5 条祝福），选择器**打乱来源、每源抽一条、取前四**——保证每次四个选项来自四种不同人格，杜绝了旧版"抽一个来源出三个同质选项"的千篇一律。选项行带来源徽记（名称+专属配色）标明性格。

### 2.8.2 五来源人格与档位纪律

| 来源 | 人格 | 配色 | 内容边界 |
|---|---|---|---|
| 灵脉余泽 | 稳妥小惠 | 玉绿 | **只做纯资源**（血/满疗/灵石/丹药），不碰牌组 |
| 补天盟密阵 | 免费改牌 | 青蓝 | 牌组操作的免费最小档（删2基础牌/升2/发2职业卡/发1金卡/蜕变） |
| 残仙遗蜕 | 付血加量 | 血红 | 每条=补天档+1 的量，代价 8–15 最大生命 |
| 照命石审判 | 极端豪赌 | 鎏金 | 法宝/破劫之印/散尽灵石换宝/250 金 -20 血 |
| 命格共鸣 | 职业优惠 | 暗紫 | class 标签过滤，同效果比通用来源便宜（身份优待） |

**核心设计纪律（血泪沉淀）**：*同类效果跨来源必须有档位差，免费版永远是最小档。* 旧版曾出现"免费升 2 张"与"扣血升 2 张"同台——付费版永无出头之日。现在的梯度：灵脉不做牌组操作（避免与补天免费档同台互殴）→ 补天免费小档 → 残仙付血大档 → 照命极端档。

### 2.8.3 特色机制：破劫之印

祝福独有的**消耗性法宝**：接下来 3 场战斗敌人以七成气血入场，用尽自碎（`battles_left` 归零时 `owner.queue_free()` 自移除）。对应 STS Neow 最著名的"前三场敌人 1 血"，是前期滚雪球的豪赌选项。技术上验证了"限次法宝"这个可复用的模式。

## 2.9 符箓丹药系统（药水位）

- 3 槽位、一次性消耗品；普通战 ~40% 掉率、精英必掉、商店上架、开局赠 2。
- 26 种覆盖：回血丹、聚气（回灵力）、抽牌符、单体/全体攻击符、上状态符、防御符等。
- **单体攻击符箓的瞄准交互**：多敌时点击进入瞄准模式（toast 提示 + 悬停亮起敌人四角框 + 右键取消），单敌一键直用——操作成本按需分级。
- 回血类战斗外可用（地图上直接嗑）。

## 2.10 法宝系统（遗物）

51 件，五类触发时机（回合开始/战斗开始/回合末/战斗末/事件驱动）。获取渠道：精英战、宝箱（二选一）、商店、祝福、事件。展示于顶栏下方专属一栏（HFlowContainer 自动换行，悬停小 tooltip——曾经的全屏点击说明已废除）。本命法宝（starter_relic）参与失败遗产循环。

## 2.11 事件系统（机缘）

45 个事件全部为"2 个实质选项 + 1 个离开"的轻量结构，选项模式集中在几类原型：付血换突破/上限、付钱换血、冒险拿钱、免费小惠。**全部数值已按 100 血再基线**（治疗 14–29、代价 6–19、金币 30–110），且修复过"文案说扣血、效果不扣血"的失同步 bug——**事件的文案数字由脚本从效果值同步生成**，此管线保证两者永不脱节。随机结果（随机突破了哪张牌）经 `ui_notice_requested` 全局 toast 播报，黑箱感清零。

## 2.12 失败遗产与元进程

死亡后记录本局获得的法宝清单（`defeat_legacy.gd`），下一局进入地图时可选一件**替换本命法宝**（或保留原本命）。这是刻意克制的 roguelite 元进程：不送数值、不解锁内容，只提供一次"带着上一世的一件遗物重新来过"的叙事连续性——失败的挫败感被转化为下一局的期待感，同时不破坏局内平衡的纯粹性。

## 2.13 卡牌关键词系统（生命周期规则）

五个关键词构成卡牌在"抽牌堆→手牌→弃牌堆"循环中的完整生命周期语法，全部在 `player_handler` 的回合结算中统一裁决：

- **消耗（exhausts）**：打出后彻底离场，本场战斗不再回到任何牌堆。用于一次性爆发牌（万祭归一/绝命爆发等 9 张），是"强力但只有一次"的定价手段。触发 `LifecycleTrigger.EXHAUSTED` 钩子——功法可以监听"每当你消耗一张牌"做引擎。
- **临时（temporary）**：英雄技能与发现机制的产物。打出即消散、回合末未打出也直接消散，**从始至终不进入牌堆循环**。与消耗的区别：消耗牌不打就留在循环里等下次抽到，临时牌是本回合限定的过客。移除时不播弃牌动画（走 remove 而非 discard 分支）。
- **固有（innate）**：开局洗牌后自动置顶抽牌堆（`move_innate_cards_to_top`）——保证首回合必在起手。用于"开局定式"类卡，给构筑提供确定性起手的设计空间。
- **虚无（ethereal）**：回合结束时若仍在手牌则按消耗处理（离场）。"用它或失去它"的紧迫感——通常配合超模数值，玩家必须为它腾出本回合的费用。
- **保留（retained）**：回合结束不弃入弃牌堆、保留在手且**保住临时降费**（`_reset_retained_card_cost` 单独处理费用归还逻辑）。跨回合蓄牌的构筑工具，与虚无正好是一对反义词。

**手牌上限**：10 张。抽牌时满手则回滚（卡牌退回抽牌堆顶）并弹出每回合一次的"手牌已满"提示（`_hand_full_notice_emitted_this_turn` 防刷屏）；英雄技能在满手时禁用（`can_use_hero_skill` 包含 `not hand.is_full()`）。上限的存在让"抽牌引擎"有了机会成本，防止无脑堆抽。

**结算顺序（回合结束弃牌阶段）**：逐张判定 临时→移除 / 虚无→消耗 / 保留→留手重置费用 / 其余→入弃牌堆，每张间隔 0.25s 保持动画节奏；全保留手牌（零清理）时跳过等待直接进入敌方回合——边界情况不卡节奏。

## 2.14 魔修构筑流派设计详解

三系统与灵根轴交叉出的四条主流构筑路线，是卡池补全与平衡工作的实际验收标准——每条流派必须"能凑、能玩、有高光时刻"。

### 流派一：血祭煞气流（自损滚雪球）

**核心循环**：主动自损（血刃/血祭斩/血盾/聚煞/命换灵机）→ 每次失血触发功法引擎（血气护体+2 护体/嗜血成性+1 回合力量/血偿万道阈值 AoE）→ 煞气随受击自然攀升 → 在 ≥6 翻涌档倾泻输出。
**关键节点卡**：聚煞（0 费自损 3 得 3 煞——最快的档位助推器）、燃脉借力类抽牌引擎补充手牌消耗。
**高光时刻**：主动把煞气顶到 10 触发天魔降世，在 ×3 回合配合 X 费"血海无量"打出全场最高单回合伤害，下回合承受 -50% 最大生命的代价——赌的就是这一回合能否终结战斗。
**弱点设计**：自损与敌方伤害共享血池，遇上猛攻型敌人（骨龙）时血线管理极其紧张；翻涌档的受伤 ×2 让"开着引擎挨打"成为最大威胁。英雄技能在此流派中是每回合稳定的失血触发器（代价 2 伤在翻涌档变 4，风险同步放大）。

### 流派二：魂印引爆流（延迟爆发）

**核心循环**：低费上印（噬魂爪/魂印/灯影噬魂/幽钩铺场）→ 功法"夺魂仪"每回合自动补印 → 攒到高层数后用裂魂/三魂同焚/魂葬引爆（每层 3 伤、吃增伤）。
**决策深度**：DoT 每层 2 伤不吃增伤、引爆每层 3 伤吃增伤——"现在收 DoT 利息还是攒本金等大引爆"随敌人血线与煞气档位实时变化。转化卡（魂铠/摄魂续元）提供防御性出口，让上印投资在被迫防守时不至于血本无归。
**高光时刻**：红·焚界四色引爆全场魂印，或天魔降世回合的 3×N×3 数字烟花。
**弱点设计**：铺垫回合多，对抗速攻阵容时可能"印还没攒够人先没了"；群怪战需要分散上印摊薄爆发。

### 流派三：七色焰轮流（回合内 Combo）

**核心循环**：收集七色魔焰卡，单回合内按最优顺序连打——每张结算时吃"已点亮其它色数"的共鸣加成。蓝·灵涌的降费与黄·狂烬的共鸣增伤是发动机，红·焚界是四色终结技。
**顺序谜题示例**：手握紫/蓝/黄/红四张时，正解通常是 蓝（回灵力+降费）→ 黄（挂共鸣增伤）→ 紫（吃 2 色共鸣+上印）→ 红（吃 3 色共鸣群伤+若达 4 色引爆全场魂印）——同样四张牌，乱序打伤害差一倍以上。
**支撑设计**：功法"焰心承续"（焰轮跨回合保留一色）与"炼焰术"（魔焰连锁降费）降低每回合从零点火的成本。
**弱点设计**：强依赖单回合手牌质量，抽不齐颜色时是全流派最疲软的；英雄技能的随机发牌在此流派价值最高（多一张可能就是多一色）。

### 流派四：灵根元素特化流（跨系统数值放大）

**核心循环**：不深耕任何魔修子系统，而是围绕所选灵根堆同元素牌——7 张圆满后全元素牌 ×1.4+1 且解锁每回合被动。火灵根的"×1.5 或 50% 溅射"选择被动与魔修的火元素卡池（血祭系多为火）天然契合，是当前版本强度最高的混合路线。
**设计意图**：给"不想学三系统"的玩家一条纯数值直觉的爬塔路线，同时它与前三流派任意杂交（灵根加成是乘在所有数值效果上的通用层）。

### 流派平衡观测点

四流派的成型速度目标：第一章末应能看出雏形（2–3 张关键卡），第二章中期成型，第三章 Boss 前达到完全体。若实测某流派"第三章还凑不齐"，优先调该流派关键卡的稀有度或抽卡权重而非数值——**成型率问题用供给解决，强度问题才用数值解决**。

# 三、界面逐屏设计

> 本章对全部界面逐一给出：布局结构、视觉与交互设计、已落地的优化清单、遗留问题与后续建议。所有界面共享的皮肤/动效规范见第四章。

## 3.1 主菜单（main_menu）

**布局**：全屏背景图（标题已烤入 `background1.png`）+ 中央透明菜单面板（继续游戏/新的轮回/图鉴大全/设置/退出，纵排）+ 四角龙纹饰件 + 顶部盘龙饰 + 左右悬挂木牌 + 底部版本牌。

**视觉与氛围**：
- 赤红灵光粒子（GPUParticles2D，160 粒，自下而上升腾，加色混合）铺满全屏——"劫气蒸腾"的第一印象。
- 背景 10 秒周期的呼吸辉光（modulate 明暗循环）。
- 暗黑修仙主视觉方向：无九宫格外框、按钮 hover/focus 态比常态略宽（expand margins），取消默认 focus 高亮。
- 入场整体淡入 0.5s + 菜单按钮错落浮现（0.07s 步进）。

**交互**：继续游戏按钮在无有效存档时禁用（`SaveGame.load_data()` 探测）；设置弹 toast 提示；按钮 hover/点击音全接入。BGM：`dark_theme_jaggedstone.ogg`（CC0 循环暗黑氛围曲），从战斗回主菜单自动切回。

**设计决策记录**：曾实现过 Ken Burns 背景漂移、鼠标视差、挂牌摇摆、龙饰呼吸光、按钮 hover 放大共五层动画，**按用户体感要求全部回滚**——主菜单保持"静中有动"的克制（粒子+呼吸辉光已足够），过多层动画反而廉价。此教训通用：**氛围界面的动效上限比战斗界面低得多**。

**遗留**：主菜单标题 logo 是烤死在背景图里的，换标题=重出背景图；设置按钮尚无实际设置面板（音量/分辨率），是 Demo 后的必做项。

## 3.2 选人界面（character_selector）

**布局**：全屏职业动画背景（按职业切换：魔修=demonic_selector_bg 帧动画）+ 左侧信息面板（名号/称谓/简介/初始血量与灵石/本命法宝）+ 底部职业卡片按钮行 + 开始修行/返回。

**技术亮点：动画背景异步加载**。职业背景是几十帧大图序列，同步加载会卡死主线程——改为 `ResourceLoader.load_threaded_request` 单飞加载 + 帧集缓存（`_frame_cache`），切换职业零卡顿；帧率可按职业覆盖（`animated_background_fps`）。

**交互与反馈**：
- 入场淡入 0.4s（只动 modulate，与异步贴图加载互不干扰——这是当初跳过此界面入场动画的顾虑，验证安全后补上）。
- 切换角色时信息面板 0.24s 淡入过渡，文字不再硬切。
- 新选中的职业卡 punch 弹跳（带 `_last_punched_index` 守卫防窗口 resize 时重复弹）。
- **开始修行按下 → 一记锣**（gong_01）：踏上旅途的仪式感锚点。

**Demo 收束状态**：当前只开放魔修，体修/剑修/驭兽按钮 disabled 但保留展示（"还有更多职业"的预告效应）。禁用态用独立的置灰卡框+暗淡文字，与可选态区分清晰。

**遗留**：初始灵石显示曾与实际不符（界面 99 vs 实际 70），已修——但这暴露了"展示常数与逻辑常数分离"的隐患，后续所有展示数值应引用逻辑侧常数。

## 3.3 灵根选择（spirit_root_selector）

**布局**：祝福石窟背景 + 居中标题"选择灵根"+ 说明行 + 三个横条选项（元素圆徽 + 名称 + 圆满效果预览文本）。

**交互**：入场淡入 0.4s、三个选项错落浮现（0.12s 步进）、hover/点击音、**选定灵根一记锣**——与选人界面的锣声连成"选人→定根"的开局仪式二连。

**信息设计**：每个选项直接展示圆满效果全文，玩家在最无知的时刻（开局）也能看到该灵根的终态收益。选项按钮用元素专属色描边（金黄/木绿/水蓝/火橙/土棕），颜色词典与卡牌元素徽记一致。

**后续建议**：加"当前职业卡池中该元素牌有 N 张"的供给量提示（见 2.4.2 构筑引导问题）；三选一之外可考虑 STS2 式的"放弃灵根换其它补偿"第四选项。

## 3.4 地图界面（map）

地图是本作在 STS 框架上视觉再创作最完整的一屏，经历了两轮大改（视觉五连 + STS 化改造）。

**布局**：居中垂直卷轴（卷轴背景按 20 层节点高度绘制，`SCROLL_FLOOR_CAPACITY=20`，当前生成 13 层）+ 两侧山景 + 顶部章节进度牌（"第X章 · 已登 Y/Z 层"）+ Run 顶栏常驻。

**路径视觉语言（杀戮尖塔化的核心）**：
- **虚线路径**：所有连线用运行时生成的 dash 纹理沿线平铺（`Line2D.LINE_TEXTURE_TILE`），替代呆板实线；
- **手绘弯曲**：每条线是 8 段采样的二次贝塞尔，弯度由端点坐标哈希决定——确定性（同一张图每次看一样）且不消耗存档 RNG；
- **四态着色**：金色=走过的路 / 暖亮+呼吸脉冲=当前可选 / 近乎隐去=已错过的支线 / 暗色=未知远路。配合"错过楼层的节点压至半灰半透明"，**来路、去路、断路一眼分明**；
- **玩家位置标记**：金色脉冲光环常驻在最后走过的节点，点击新节点时在 select 动画窗口内滑行过去（带跳跃感），本章首选原地弹出。

**节点交互**：
- 可选节点常态呼吸发光（不 hover 也知道能点）+ hover 放大 1.18 回弹 + 轻音；
- 每类节点 hover/focus 显示含义 tooltip（战斗/精英/宝箱/营火/商店/机缘/赐福的说明+"点击前往"状态行）——新玩家零学习成本；
- **Boss 节点常驻红光脉冲**（仅 Boss 开 `_process`，无全图开销）——从地图底部即可感知顶端威慑，与战斗内 Boss 名牌横幅形成连续的压迫叙事。

**运动手感**：滚轮从步进跳变改为目标值插值的平滑滚动（步长 ×2.4 但过程丝滑）；开图时节点自下而上按层错落浮现 + 连线整体淡入（仅新章/读档触发，战斗返回不重播）。

**遗留**：地图背景静态，视差/云雾飘动需分层美术资产；FLOORS 仍为 13，卷轴已预留到 20 层，做 20 层章节只改一个常数。

### 3.5.2 信息卡设计（battle_combatant_card）

战斗单位统一用信息卡呈现：顶部名称、中央大立绘（左右仅留条宽边距——立绘占满卡窗是压制"黑盒感"的关键）、左缘竖置血条、右缘竖置护体条、**条底数字横排**（血条数字向卡内右伸、护体向卡内左伸——向外伸会被 clip_contents 裁切，这是修过的真实 bug）、底部状态图标行（32px 图标+层数，带边框 chip 与悬停 tooltip）。敌卡顶上悬挂意图气泡（分类图标+文字+数值），玩家卡顶上悬挂煞气徽章。

**数值动效**：血/护体条 0.45s tween 过渡 + 掉血闪白/加盾闪蓝 + 数字 punch；状态图标签名比对刷新（新词条弹入、层数变化 punch）；意图变化徽章弹跳闪亮。组件有 0.12s 轮询兜底，**所有动画必须走 diff 检测**否则活不过一帧（架构约束，已写入交接文档）。

**关键工程决策：世界节点对齐信息卡**。瞄准碰撞、选中框、飘字锚点原生长在（已隐藏的）世界立绘节点上，与信息卡位置脱节曾导致"选中框和怪对不上"的严重 bug。解法是布局后把每个战斗单位的世界 Area2D **反向对齐到其信息卡中心**（画布坐标反变换），碰撞体与四角选中框放大到卡尺寸——此后一切定位型反馈（飘字/特效/瞄准/丹药点选）天然落在卡上。**新增任何战斗内定位逻辑必须考虑这层对齐。**

### 3.5.3 手牌交互状态机

CardUI 五状态：Base（扇形定位）→ Clicked → Dragging（自由拖拽/瞄准预览）→ Aiming（单体攻击弧线瞄准）→ Released（结算）。手感参数全部常量化：

- **扇形布局**：卡 224×322、弧形下沉、中央上提；hover 聚焦 0.15s 快速响应 + TRANS_BACK 回弹放大 1.38 + 邻牌按距离衰减让位（46px×0.45^距离）；
- **补位弹簧**：牌数变化的重排用 TRANS_SPRING 0.48s（与 hover 让位的快速 cubic 分离——两种场合两种手感）；
- **拖拽惯性**：按鼠标横向速度倾斜 ±9°，停驻 0.4s 回正；
- **瞄准吸附**：弧线终点锁定敌人中心，锁定时弧线/箭头红→亮金渐变+箭头 pop，敌方四角框回弹收拢+呼吸脉冲；
- **可打出辉光**：费用足够的牌带暖金外发光（alpha 0.20），费用不足点击→卡面左右摇头+费用数字闪红+错误音；
- **牌流动画**：抽牌从抽牌堆飞入（0.3 缩放+淡入+弹簧归位）、弃牌以幽灵卡飞向弃牌堆（不动真卡状态机，零输入风险）、洗牌时幽灵卡背从弃牌堆错落飞回抽牌堆+计数 punch、出牌中央放大悬停预览后淡出、**功法牌特殊演出**——化金光飞向角色融入+金色爆发（永久牌与一次性牌的视觉区分）。

### 3.5.4 反馈系统全家桶

以"伤害落地那一帧"为锚的多通道反馈（当前节奏为三轮调速后的最终版）：

| 通道 | 内容 | 关键参数 |
|---|---|---|
| 飘字 | 伤害红-52号/格挡蓝"盾N"-42/自损暗红-44/治疗绿+42 | 弹出过冲1.25倍→停留1.3s→上浮淡出1.2s，描边9px |
| 命中特效 | 代码绘制放射冲击线+中心闪光 | 0.55s；敌人橙金/玩家红/功法金 |

### 3.5.5 遗留与建议

- 攻击命中缺一帧"斩击特效贴图"（当前是代码放射线），美术到位后可升级；
- 世界立绘（魔修 5 套帧动画）在信息卡制下被隐藏——"战场化"方案（立绘站桩+紧凑名牌）作为长期备选保留在设计记录中，用户当前选择维持信息卡制；
- 多段连击时飘字可能堆叠，若实测挡视线可把 HOLD 缩至 1.0s。

## 3.6 战斗奖励（battle_reward）

**结构**：暗幕+居中面板，奖励按钮纵列错落浮现（0.09s 步进）：灵石（点击金币音+顶栏数字滚动）→ 择取新术法（进入三选一子界面）→ 符箓丹药（概率/精英必掉）→ 精英另有法宝与"术法熔炼"（融合）。

**卡牌三选一子界面**：三张 CardMenuUI 错落弹出，hover 浮起 1.05+抬 z 序，点选看 tooltip、"领取"确认——**领取的卡从卡位飞向顶栏总牌库按钮 + 计数 punch**（与商店购卡共用同一条 `card_acquired_animation_requested` 动画管线，全游戏"获得卡牌"的视觉语言统一）。可跳过（放弃拿牌是构筑纪律的一部分，永远可用）。

**建议**：跳过卡牌可补偿少量灵石（STS2 方向），提高"瘦牌组"策略的正反馈。

## 3.7 商店（shop）

**结构**：掌柜立绘+三排货架（卡牌 ×N / 法宝 / 丹药）+ 删卡服务 + 返回。货架物件错落浮现（0.07s 步进）。

**交互与反馈**：价格按稀有度色阶显示、灵石不足时价格转红+购买钮禁用；购买瞬间铜钱音+顶栏灵石滚动扣减；**买卡后幽灵卡飞向总牌库**；优惠券法宝挂商店折扣钩子（`add_shop_modifier`）。暗金卡 4% 概率上架制造逛店期待。

**遗留**：无刷新货架机制（可作付费 reroll 扩展）；删卡定价未随次数递增（STS 是 75+25/次的通胀模型，值得引入）。

## 3.8 宝箱（treasure）

**结构**：居中宝箱立绘，点击开箱——木箱开启音+宝石音+爆光粒子演出（`_begin_open_effects` 的进度驱动绽放）→ 进入**法宝二选一**（复用奖励界面的选择流）。二选一是从"随机送一件"改来的：宝箱从彩票变成了决策点。

## 3.9 篝火（campfire）

**结构**：篝火场景+两键：休整（回 30% 最大生命，治疗音+飘字）/ 突破（进入选卡升级子界面，升级书页音）。选择后淡出返回地图。界面入场淡入、面板 InkTheme 统一皮。

**建议**：第三选项空间（"回忆"看牌库统计 / "铸魂"移除一张牌）是营火决策丰富化的经典方向。

## 3.10 事件房（机缘，45 个场景）

**结构**：通用模板（generic_event）——标题+正文+插图（`art/event_illustrations/`）+ 最多三个选项按钮；少数专属脚本事件（赌局、热心小妖）。入场淡入统一在 `run.gd` 进房时施加（事件场景众多，不逐个改 `_ready`——集中式挂载的运维经验）。

**信息与反馈纪律**：选项文案数字由脚本从效果值生成同步（修过文案效果不符 bug）；无法支付的选项（灵石不足）禁用；随机结果（突破/移除了哪张牌）toast 播报。

## 3.11 祝福界面（劫中遗赠，blessing）

**结构**：石窟背景+标题"劫中遗赠"+章节说明行+对话条（"数道先古回响同时低语，只可应其一"）+ 四个横条选项（来源徽记[配色] + 祝福名 + 效果全文），错落浮现，选定一记锣。

**信息设计**：来源徽记的颜色即风险等级的直觉编码（绿=安全…金=豪赌），效果全文直白展示包括代价——祝福是开局最重的决策，**零信息隐藏**。

## 3.12 图鉴（codex）

**结构**：左侧目录树（总览/卡牌[按职业二级分组]/法宝/怪物/词条/符箓丹药，各带计数）+ 右侧详情面板（图标+名称+元数据+描述+实卡预览）+ 总览页的资源统计行。全屏独立场景，主菜单进入。

**技术**：运行时 `DirAccess` 递归扫描资源目录——新增同类资源自动入册，新资源类型需手动加扫描根（架构约定）。

**交互**：条目切换详情面板 0.18s 淡入 + **书页翻动音**（翻书音与图鉴场景的主题契合是刻意选择）；预览卡自动获得 CardMenuUI 的弹入+hover 浮起。

**遗留**：无搜索/筛选（内容量 300+ 后必须）；词条与卡牌之间无交叉跳转链接。

## 3.13 牌组操作四件套（牌堆查看/删卡/升卡/融合）

统一用 CardMenuUI 网格：总牌库（顶栏按钮，战斗中挂独立 DeckViewLayer 防止与 HUD 错位——修过的层级 bug）、抽牌堆（随机排序显示防记牌）、弃牌堆、删卡（炼化）、升卡（突破，带前后对比）、融合（配方匹配高亮）。全员享受组件级的错落弹入与 hover 浮起；打开即 0.2s 淡入（高频操作用最快的入场档）。

## 3.14 卡牌发现界面（Discover Overlay）

**机制**："发现"是运行时三选一拿临时牌的效果（`configured_discover_effect` → `card_discovery_requested` 信号 → battle_ui 的全屏遮罩 overlay）。与炉石"发现"同源：从一个池子里展示若干张，玩家挑选加入手牌（临时牌属性，不入循环）。

**界面结构**：半透明暗幕（阻断战斗输入）+ 横排候选卡（CardMenuUI，自动获得弹入与 hover 浮起）+ 已选计数标签 + 确认按钮。选择数量由请求方定义（支持多选场景）。

**设计定位**：发现是"运行时构筑"的入口——牌组之外的即时资源，与英雄技能的随机发牌共同构成魔修的"手牌之外还有牌"体验层。因为产出全部是临时牌，不会污染牌组构筑的长期纪律。

**约束**：overlay 打开期间战斗输入必须全量阻断（手牌禁用、技能与结束回合置灰），关闭后恢复——模态层的输入纪律与暂停菜单一致。

## 3.15 全局通用组件

- **通知 toast（NoticeLayer）**：run 层 21 号 CanvasLayer 的居中堆叠通知，逐条淡入-停留 2.4s-淡出。全游戏随机结果播报（祝福/事件随机到的牌、丹药瞄准提示、手牌已满）共用此管线。深色面板+棕金描边与 InkTheme 一致。
- **法宝 tooltip**：悬停小浮窗（曾是点击全屏说明，已废除——高频查询用轻量交互）。
- **卡牌 tooltip 弹窗**：牌组操作界面点选卡牌的说明浮层，带图标与完整描述。
- **DeckViewLayer**：战斗中查看总牌库挂 18 号独立层，解决与战斗 HUD 的层级错位（修过的真实 bug——模态视图必须有自己的 CanvasLayer）。

## 3.16 暂停菜单 / 胜负结算 / 胜利画面

- **暂停**（ESC/齿轮）：暗幕+居中面板缩放回弹入场（树已暂停，tween 用 `TWEEN_PAUSE_PROCESS`——所有暂停态动画的通用约束），继续/返回山门。
- **战斗胜负面板**：整体淡入+标题回弹（胜利从大缩小、失败从小放大——两种情绪两种曲线）+按钮延迟浮现。
- **胜利画面**（win_screen）：三章通关结算，0.45s 从容入场。**遗留**：内容单薄（无本局统计——用卡次数/伤害总量/构筑回顾），是"值得炫耀的结算"的必做扩展。

# 四、UI 美术设计规范

## 4.1 视觉主题：暗黑修仙（Dark Guofeng）

**主题词**：劫、墟、残仙、黑金。视觉基调是"辉煌文明的废墟"——金色代表曾经的仙道秩序（边框/文字/走过的路），暗红与紫代表劫气与魔染（魔修主色/危险提示/Boss），深蓝黑是劫域的夜（背景/面板底色）。

**色彩词典**（核心色值，全项目复用）：

| 语义 | 色值 | 用途 |
|---|---|---|
| 主金 | `f2c94f` | 标题、选中态、我方回合横幅、走过的路径 |
| 暖字金 | `f8e5c0` / `fff0c2` | 按钮文字、正文亮字 |
| 危险红 | `e0503c` / `d0342c` | 敌方回合、煞气高档、代价文本 |
| 魔紫 | `b04ae0` / `b88ad8` | Boss 名牌、命格共鸣、魔焰视觉 |
| 元素五色 | 金`d9c77a` 木`79b66a` 水`6fb2d8` 火`e06a3b` 土`b99358` | 灵根/卡牌元素徽记全局一致 |
| 面板底 | `Color(0.03~0.05, 0.02~0.03, 0.02~0.03, 0.82~0.94)` | 一切深色面板 |
| 描边棕金 | `Color(0.58~0.74, 0.38~0.54, 0.20~0.26)` | 面板/按钮边框 |

**稀有度色阶**：白 `e8e1cc` / 蓝 `4c9fd6` / 金 `d7a93f` / 暗金 `7a3a18`——暗金刻意用焦褐而非亮橙，符合"劫火淬炼"的题材气质。

## 4.2 InkTheme 设计系统

`scenes/ui/ink_theme.gd` 是全项目唯一的皮肤中心，**任何新界面禁止自造 StyleBox**。提供的能力分四层：

1. **原子样式**：`make_style()`（圆角面板：底色/描边/阴影全参数化）、`make_texture_style()`（9 切贴图面板）。
2. **组件套装**：`apply_screen_button/secondary/danger`（三档语义按钮）、`apply_battle_blue_button`（战斗蓝金大按钮）、`apply_demonic_button`（魔修红金）、`apply_screen_title/subtitle`（标题层级）、`apply_screen_panel`、图标底板等。
3. **行为注入**：`wire_button_sfx()`——凡走 apply_screen_button 的按钮自动获得 hover/点击音（meta 标记幂等）；自定义样式按钮手动调一行接入。**声音是设计系统的一部分，不是逐界面的散活。**
4. **动效原语**：`animate_screen_entrance(root, duration)`（界面级淡入，全部 14+ 界面接入：常规 0.3s、高频操作 0.2s、氛围场景 0.4–0.5s）、`animate_item_entrance(item, stagger)`（列表项按容器下标错落淡入，封顶 20 项——奖励按钮/商店货架/菜单按钮通用）。

**字体层级**（当前用系统字体，国风正式字体是待办）：屏幕大标题 48–58 / 界面标题 35–44 / 按钮 21–25（战斗大按钮 36）/ 正文 20–25 / 辅助信息 13–18。所有文字带 shadow（offset 2–3，黑 0.7–0.95 alpha）保证深色背景可读性。

## 4.3 卡牌视觉规范

- **卡框七系**（`art/ui/cards/generated/`）：雷紫/剑金/心法青玉/秘术暗紫/丹道橙金/剑阵赤红/默认古金——按卡牌类型映射（card_style.gd）。**插画窗已洪水填充挖空为透明**，渲染顺序"插画在下、金边压上"，插画铺满窗口。新卡框必须保持相同窗口区域与透明窗，否则牵连整套 UI。
- **锚点比例布局**（card_visuals.tscn）：所有元素按比例锚定，任意尺寸不漂移——同一套视觉组件服务战斗手牌（224×322）/商店/奖励/图鉴预览/删升融全部场合。
- **动态字号**：卡名按长度 14–18、描述按字数 10–13 自动降档，杜绝溢出。
- **状态区分**：hover 提亮框+高光阴影 22px、拖拽更亮 28px、可打出常态金辉光 alpha 0.20、禁用整体置灰+费用转红。
- **插画规范**：剑修/魔修已接 AI 生成大插画（`art/cards/<职业>/ai/`），魔修新卡出图必须参考现有人物风格避免卡面角色长相割裂（HANDOFF 明令）；体修/驭兽仍是 16px 占位。
- **全量 mipmap 纪律**：一切会缩小显示的高清图开 mipmap（消除缩放白点），**16px 像素占位图严禁开**（会糊）。

## 4.4 动效设计语言

**时长基线（第三轮调速后的现行标准）**：

| 类别 | 时长 | 曲线 |
|---|---|---|
| 微交互（hover/punch） | 0.14–0.4s | CUBIC out / BACK out（回弹） |
| 布局重排 | 0.24s（hover 让位）/ 0.48s（补位弹簧 SPRING） | 分场合双轨 |
| 界面入场 | 0.2/0.3/0.4–0.5s 三档 | CUBIC out |
| 战斗飘字全程 | ~3s（弹出0.45+停留1.3+淡出1.2） | 弹出 BACK 过冲 1.25 |
| 横幅 | 入 0.2/0.32 + 停 0.8–2.4 + 出 0.36 | 重要度决定停留 |
| 循环呼吸 | 0.55–1.5s 半周期 | SINE in-out |

**三条铁律**：
1. **多循环动画的周期必须错开**（主菜单粒子/辉光、地图脉冲/呼吸各不同步）——同步呼吸=机械感；
2. **调速用统一比例缩放**——本项目经历"太快→放慢35%→飘字再放慢一倍"三轮，每轮都是全体常量等比缩放而非零敲碎打，节奏关系始终保持；
3. **暂停态动画必须 `TWEEN_PAUSE_PROCESS`**，轮询刷新组件里的动画必须 diff 检测。

## 4.5 音频设计

**结构**：`SoundPlayer`（8 路 SFX 复音+每路音量参数）+ `MusicPlayer`（4 路）+ `GameSfx` 静态音效库（全部预载、变体数组随机——**刻意不用 RNG autoload，音效不消耗存档种子**）。

**授权纪律**：当前音频 100% CC0（Kenney/rubberduck/HaelDB/nene/Wolfgang_ 等，来源与授权登记在 `THIRD_PARTY_AUDIO.md`），CC-BY 素材刻意未用——零署名义务。新增音频必须先过授权登记。

**映射与混音基线**（-2~-16dB 按触发频率递减）：卡牌操作（滑动/落卡/推挤/洗牌）、打击三件套（hit/slam/metal）、国风重音三件套（**锣=煞气跨档与开局仪式、钟=Boss 名牌、书页=图鉴翻阅与升卡**——国风乐器只用在"命运时刻"，日常操作用中性音，保持重音的稀缺性）、UI 通用（tap hover -14 / 落卡点击 -8 / error）。

**BGM 策略**：主菜单暗黑氛围曲 → 普通/精英战紧张管弦 → Boss 战按敌群 id 自动切交响金属（.import 已开循环）。**空缺**：地图/商店无独立 BGM（战斗曲延续），备选 `vampires_piano`（CC0）待接；国风 BGM 需从 Pixabay 手动下载（其授权禁止镜像分发）。

## 4.6 资产管线

AI 出图工作流暂存于 `tmp/`（已 gitignore）；9 切资产统一 `assets/ui/generated/` 与 `art/ui/`；**新增/替换 PNG 必须跑一次编辑器导入**（`--headless --editor --quit`），否则运行时"文件在但加载失败"且 validate 查不出——这是本项目最高频的坑，交接文档列为一级注意事项。

# 五、游戏逻辑架构与调整记录

## 5.1 场景与信号架构

**Autoload 五件套**：`Events`（全局信号总线）/ `RNG`（种子化随机，存档记录 seed+state 保证复现）/ `Shaker`（震动，恢复位置用 instance_id 反查）/ `MusicPlayer` / `SFXPlayer`。

**Events 总线**是全项目的解耦中枢，信号按域分组：卡牌（drag/aim/played/drawn/discarded/discovery…）、玩家（turn_started/hit/self_damaged/hand_drawn…）、敌人（action_completed/died）、战斗（won/over_screen）、英雄技能（requested/used）、经济与房间（shop/campfire/treasure/event/blessing）、表现层（flame_wheel_changed/sha_qi_tier_changed/deck_reshuffled/card_acquired_animation_requested/ui_notice_requested）。**规则**：跨场景通信一律走 Events；连接方负责在 `_exit_tree` 对称断开；表现层信号（动画/toast）与逻辑信号分离命名。

**Run 主控**：`run.gd` 持有地图/顶栏/法宝/丹药/存档/当前视图容器，房间进入=实例化子场景挂到 CurrentView；战斗胜利→按房间类型分发奖励流；Boss 后推进章节回满血。战斗则由 `battle.gd` 编排 player_handler（回合与牌流）/ enemy_handler / 三个机制处理器（灵根/职业机制/法宝）。

## 5.2 关键时序与工程约定（事故沉淀）

1. **攻击动画门控**：攻击牌→播动画→`attack_animation_finished` 才结算伤害；受击同理（动画播完扣血）。任何绕过此门控的直伤会导致"数字先跳动画后打"的脱节。
2. **敌人延迟回调必须捕获 `instance_id`**：补间 finished 回调若直接捕获节点，敌人先死会 `Lambda capture freed`。已根治 7 处，run-flow 仍有一处偶发来源未定位（技术债）。
3. **世界节点对齐信息卡**（3.5.2 详述）：战斗内一切"位置"以信息卡为准。
4. **modifier 原地改值**：阈值型修正（煞气档位）复用同一 ModifierValue 改数值，禁用 remove_value（queue_free 延迟会丢值）。
5. **HitPause 无头跳过**：time_scale 操作会拖慢 smoke 测试的计时器（曾把 boss-battle 推过 60s 超时线）。
6. **卡牌关键词生命周期**（近期演化完成的一组 STS 关键词）：**临时**（不入循环，回合末消散）/ **固有**（innate，开局置顶抽牌堆）/ **虚无**（ethereal，回合末未打出则消耗）/ **保留**（retained，回合末不弃且保费用）/ **消耗**（exhausts）——统一在 `player_handler.discard_cards` 的分支结算，配合 `LifecycleTrigger` 钩子。手牌上限 10 张（超限抽牌回滚+每回合一次的满手提示）。

## 5.3 数据管线（策划面）

- **`game_data.xlsx`（根目录）三表**：卡牌（费用/伤害/护体/层数等标准数值）/ 祝福（描述/命格/效果+值，导入整表重建 JSON）/ 怪物AI（4 教学敌人的数值与出招序列）。工作流：改浅黄列 → `xlsx_to_game_data.py` 回写 → Godot 导入。**边界**：魂印引爆倍率、魔焰 color/rider 等机制字段表中留空、只能在 Godot 改——防止表格把机制改坏。幂等保证：无修改回写=改 0 文件。
- **`data/blessings.json`**：祝福全量数据，新效果类型需同时登记 `blessing.gd._apply_effect` 分支 + `blessing_table.py EFFECT_TYPES`。
- **事件**：数值内联在各 .tscn 的导出数组，文案数字由脚本从效果值同步生成。

## 5.4 存档系统

`SaveGame` 记录：RNG seed+state、角色与牌组、血量、法宝、丹药、地图数据与进度、章节、位置。**自愈机制**：加载前 `get_dependencies` 检查依赖完整性，坏档自动清理返回主菜单（不崩不卡）。**已知架构隐患**：资源按路径内嵌引用，移动/重命名被引用资源=旧档失效（有自愈兜底但丢进度）——根治方案"按 id 重建"列为 ROADMAP 阶段 0。

## 5.6 Events 信号总线速查

跨场景通信的全部契约（按域分组，括号内为主要消费者）：

**卡牌流**：`card_drawn/card_played/card_discarded/card_exhausted/deck_reshuffled`（battle_ui 动画与音效、class_mechanic_handler 机制侦听、player_handler 入弃牌堆）、`card_drag/aim_started/ended`（瞄准器与手牌互斥锁）、`card_play_preview_requested`（中央预览）、`card_discovery_requested`（发现 overlay）、`card_extra_drawn`（非回合初抽牌的引擎钩子）。

**玩家流**：`player_turn_started/ended`（回合横幅、机制重置、技能限次刷新）、`player_hand_drawn/discarded`（回合门控）、`player_hit`（煞气获取）、`player_self_damaged`（自损飘字与引擎）、`player_died`、`attack_animation_finished`（攻击伤害门控）。

**敌人流**：`enemy_action_completed`（敌方回合推进）、`enemy_died`（胜利判定）、`enemy_turn_ended`。

**表现层专用**：`flame_wheel_changed`（焰轮盘刷新）、`sha_qi_tier_changed`（煞气演出）、`hero_skill_requested/used`（技能触发与限次）、`card_acquired_animation_requested`（获卡飞牌库，商店/奖励共用）、`ui_notice_requested`（全局 toast）、`tooltip` 族。

**房间流**：`battle_won/battle_over_screen_requested`、`map_exited`、`shop_*_bought/shop_exited`、`campfire_rested/card_upgraded/exited`、`treasure_room_exited`、`event_choice_resolved/event_room_exited`、`blessing_exited`。

**使用纪律**：表现层信号（动画/音效/toast）与逻辑信号分开命名，禁止逻辑依赖表现层信号的时序；连接方 `_exit_tree` 对称断开；新增信号先查有无可复用的既有信号（`card_acquired_animation_requested` 被商店与奖励复用即是范例）。

## 5.7 测试与验证体系

## 7.3 设计原则清单（本项目沉淀）

1. **同类效果跨来源必须有档位差，免费版永远是最小档**（祝福支配关系事故）。
2. **随机效果必须播报结果**——"随机突破了哪张牌"不能是黑箱（ui_notice 管线）。
3. **文案数字与效果值同源生成**，禁止手写两份（事件文案失同步事故）。
4. **机制可见性三件套**：常驻状态（徽章）+ 瞬时宣告（横幅）+ 持续氛围（气场染色），分别回答"多少/刚发生什么/大致状态"。
5. **代价的语义要选定并贯彻**："失去生命"（不可防）与"受到伤害"（可防、吃修正）是两种设计，文案与机制必须一致。
6. **反馈以结算帧为锚**，多通道（视觉/听觉/触觉替代）同时到达；节奏参数集中常量化，调速走等比缩放。
7. **国风重音（锣/钟/书页）只用于命运时刻**，日常操作用中性音——稀缺性即仪式感。
8. **一切界面走 InkTheme**，一切跨场景通信走 Events，一切新可收集资源接图鉴扫描。
9. **敌人强度三层分离**（基线/每场/章节），调平不跨层补偿。
10. **氛围界面的动效上限低于战斗界面**（主菜单五层动画回滚事故）——动效服务信息，不服务炫技。

# 八、战斗手感工程全录

> 本章完整记录 2026-07 手感优化专场及其后续三轮迭代的全部条目——每条包含设计动机与关键参数，作为后续调参与新增反馈的对照基准。手感工程的总原则见 1.4 支柱二与 4.4 三条铁律。

## 8.1 手牌交互组（6 项）

**① Hover 聚焦**：响应 0.15s（比布局重排的 0.24s 快一档——手指下的东西必须最先动），TRANS_BACK 回弹放大 1.38 倍，聚焦卡转正消除扇形倾角。动机：STS 的"卡牌迎手"感，放大幅度要足以读清描述全文。
**② 邻牌让位**：聚焦卡两侧的牌按距离衰减推开（46px × 0.45^距离）——放大的卡不压邻牌，同时让位量随距离衰减避免整排牌大幅晃动。
**③ 补位弹簧**：牌数变化（抽/打/弃）引起的重排用 TRANS_SPRING 0.48s——牌与牌之间"挤"出来的弹性；与 hover 让位的快速 cubic 严格分轨，两种物理感不混用。
**④ 拖拽惯性**：自由拖拽按鼠标横向速度累积倾斜（每像素 0.14°，上限 ±9°），停驻 0.4s 摆正。动机：卡牌是"有质量的纸"，不是贴着光标的贴图。
**⑤ 费用不足摇头**：点击打不出的牌→卡面子节点横向衰减摆动（10/-8/6/-3/0 像素序列，每步 0.06s）+ 费用数字闪红 0.6s + error 音。摇的是视觉子节点而非 CardUI 本体——不与布局 tween 抢属性。
**⑥ 可打出辉光**：费用足够的牌常态带暖金外发光（alpha 0.20，曾用 0.32 因整排手牌泛白雾而调低）——"哪些牌能打"变成被动可见信息，不需要逐张试。

## 8.2 牌流动画组（5 项）

**⑦ 抽牌飞入**：新卡从抽牌堆按钮位置以 0.3 缩放、-14° 倾角、透明态起飞，被布局弹簧接住归位；起点坐标由 battle_ui 布局时写入 hand（`draw_origin_global`），窗口缩放自动跟随。
**⑧ 弃牌幽灵**：回合结束弃牌时真卡立即释放，克隆一张轻量 CardVisuals 幽灵卡飞向弃牌堆（0.44s 缩小旋转淡出）。用幽灵而非真卡的原因：CardUI 挂着输入状态机，动画期间的残留输入是事故源——**表现与逻辑对象分离**。
**⑨ 洗牌提示**：弃牌堆洗回抽牌堆时，至多 4 张空白卡背从弃牌堆错落（0.09s 步进）飞向抽牌堆 + 计数 punch。动机：洗牌原本是完全无提示的暗改，玩家会困惑"我的弃牌堆怎么清零了"。
**⑩ 出牌预览**：打出的牌飞至战场中央放大 1.45 悬停 0.6s 再淡出——给"我刚打了什么"一个确认帧，联机观战式的可读性。
**⑪ 功法金光**：POWER 牌的预览结束后不淡出，而是化金光飞向角色融入 + 金色爆发。动机：永久生效的牌值得一个"入体"仪式，与普通牌的"用完即走"在视觉上分家。

## 8.3 打击反馈组（7 项）

**⑫ 飘字系统**：四色语义（伤害红/格挡蓝/自损暗红/治疗绿），画布坐标锚定不受世界缩放影响；伤害在结算帧按当前护体拆成"扣血红字+盾字"两条——玩家能看到这一下被挡了多少。三轮调速后终态：52 号大字、过冲 1.25 弹出、停留 1.3s。
**⑬ 命中特效**：代码绘制的 7 根放射冲击线+中心闪光（0.55s），随机角度/长度/宽度避免复读感；挂在受击者父层级——受击者死亡释放不影响特效播完。
**⑭ 命中停顿**：实际扣血瞬间 time_scale 压至 0.05，敌方 80ms/玩家 120ms（玩家挨打更重）；连续命中取最长窗口不叠加；恢复计时器 ignore_time_scale。
**⑮ 震动**：受击者位置抖动（衰减式 10 次）+ 玩家挨打附加战场根节点 3px 微震——屏幕震动保留给"我受伤了"，敌人挨打不震屏（保持打击方向感）。
**⑯ 敌人攻击特效**：敌人出攻击类招式瞬间在其信息卡上闪红色冲击（代替曾经的位置突进——世界立绘隐藏后移动节点不可见的教训）。
**⑰ 死亡溶解**：敌人染红+横向压扁+下沉+淡出 0.62s 后才真正释放与胜利判定；溶解中关碰撞、藏意图、锁 do_turn。硬约束：总时长 ≤0.8s（boss smoke 击杀后 1.0s 断言胜利面板）。
**⑱ 治疗与 DoT 飘字**：治疗经"血量增量检测"统一挂绿字（丹药/转化/灵根被动/敌方回血全覆盖，单点钩子）；DoT 因走标准 take_damage 管线自动获得全套反馈——**管线复用优于逐处挂接**。

## 8.4 回合与状态演出组（7 项）

**⑲ 回合横幅**：敌我回合切换中央横幅（金/红），弹入-停留 0.8s-上浮淡出，重复触发打断重放。**⑳ Boss 名牌**：开场 1.45s（等回合横幅播完）紫色名牌+钟声，停留 1.6s。**㉑ 煞气跨档**：升档横幅+锣、降档只静默回退气场——涨要宣告、跌不打扰。**㉒ 意图弹跳**：敌人换招时意图徽章 1.25 倍弹跳+闪亮。**㉓ 焰轮点亮**：新色珠 1.7 倍放大回落+高亮闪（黑色魔焰提亮兜底）。**㉔ 结束回合呼吸**：无牌可打时按钮亮度呼吸（0.2s 轮询检测手牌可打状态）。**㉕ 状态图标生命周期**：新词条弹入、层数变化 punch、签名比对防轮询重建杀动画。

## 8.5 参数调音史（三轮全局调速）

1. **第一轮（初版）**：以 0.11–0.42s 的"敏捷"节奏落地全部 25 项——实测反馈"动画有点快"。
2. **第二轮（全局 ×1.35 放慢）**：所有时长常量等比放大约 35%，节奏关系不变——布局 0.18→0.24、飘字全程 0.74→1.05s、横幅停留 0.56→0.8s。
3. **第三轮（飘字定向 ×2 再放大）**："伤害数字看不清"的定向反馈——仅飘字通道再翻倍并加大字号/描边/过冲，其余通道维持第二轮节奏。
**方法论**：全局体感问题用等比缩放，单通道问题做定向调整；每轮调速后跑截图对照+全量 smoke，参数变化必须留档（HANDOFF"节奏基线"条目）。

# 九、STS2 卡牌数据研究结论（卡池方法论来源）

> 本章存档对《杀戮尖塔2》v0.107.x 全部 606 张卡的量化分析结论。魔修卡池的补全（2.1.3）直接执行了这些发现。

## 9.1 六卡池总量与预算的"工业化对齐"

五个可玩职业每人 87–89 张，卡数差异不超过 2%；稀有度配额逐职业**完全相同**（罕见 36/稀有 26/普通 20/基础 3–4/远古 2）。结论：顶级卡牌肉鸽的职业差异化完全发生在机制层，数量与稀有度预算是先行锁死的模板——"先定预算，再填主题"。本作执行：魔修 67 张按白 28/蓝 23/金 11/暗金 5 的金字塔配比。

## 9.2 类型比与费用曲线的语义

全局技能 297 > 攻击 197 > 能力 112。技能承载引擎（过牌/资源/防御），能力牌稀缺（每职业约 19 张）且几乎全部是"每当 X→Y"触发器——**能力牌定义 build，数量稀缺维持其决策重量**。费用曲线：1 费主干（44–52 张/职业）、0 费做节奏、3 费+刻意稀缺、每职业雷打不动 2 张 X 费。升级哲学统一：数值 +1~2 档、或降费、或去消耗。

## 9.3 三种职业复杂度范式

- **单核深潜**（故障机器人）：88 张里 44 张围着充能球一个系统，生成/扩容/激发/回合触发四个动词做穿。
- **多线并行**（静默猎手）：中毒/小刀/敏捷/弃牌四条浅线各自成立、互不依赖。
- **多资源耦合**（亡灵契约师）：宠物/灾厄/灵魂/虚无四资源交叉，复杂度最高。
魔修选择了第三范式（煞气/魂印/焰轮），并以 2.13 的四流派验收表证明耦合点足够产生流派分化——这是选择该范式的兑现标准。

## 9.4 对本作的五条直接移植

① 专属命名资源做职业锚点；② 功法占比 ~20% 且全部是触发器；③ X 费恰好 2 张封顶；④ "消耗"关键词从 2 张扩到 9 张（献祭题材本就该重用一次性爆发）；⑤ 攻击前倾档位（45/36/19）对齐铁甲战士而非全局均值——类型比服从职业性格而非平均主义。
